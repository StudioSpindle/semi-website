Charts

We developed / adapted a number of useful charts to visualise data and insights from SeMI. They are provided in a small chart library. The examples below are simple use cases of the charts. All charts are interactive with at least a tooltip. The demo data used in the examples below can be found in semi-demo-data.js

Please note that the snippets need to be pasted in an environment where the SeMI CSS is loaded.

Sankey Diagram

The Sankey Diagram is useful for showing the connectivity in a graph, i.e. to show through which paths nodes are connected.

Chart Options

width
Width of the chart.
Default: 600
height
Height of the chart.
Default: 600

Bubble Chart

The Bubble Chart was also used in the Playground experiments for showing the connectivity in a graph, i.e. show the number of connections between one node category to another.

Chart Options

width
Width of the chart.
Default: 600
height
Height of the chart.
Default: 600
top
Padding top.
Default: 30
bottom
Padding bottom.
Default: 80
left
Padding left.
Default: 100
right
Padding right.
Default: 80
xCategory
Name of the object property being mapped to the x-Axis.
Default: ''
yCategory
Name of the object property being mapped to the y-Axis.
Default: ''
valueCategory
Name of the object property being mapped to the bubble size.
Default: ''

Force-Directed Graph

The Force-Directed Graph is a graph visualisation based on d3-force. Tooltips display all node properties.

Chart Options

width
Width of the chart.
Default: 600
height
Height of the chart.
Default: 400
nodeRadius
Radius of each node.
Default: 5
colorBy
Name of the object property which is used for creating categories. The colors are assigned by category.
Default: 'class'

Constraint-Based Layout

The Constraint-Based Layout is a graph visualisation which groups nodes together by category and colors them accordingy. Tooltips display all node properties.

Chart Options

width
Width of the chart.
Default: 600
height
Height of the chart.
Default: 400
nodeRadius
Radius of each node.
Default: 5
nodeSpacing
Value indicating how much space around each node is tried to be preserved.
Default: 20
showGroups
Shows a rectangular border around each category group if true.
Default: false
colorBy
Name of the object property which is used for creating categories. The colors are then assigned by category.
Default: 'class'

Beeswarm Chart

The Beeswarm chart is a mix between the Bubble Chart and the Graph visualisations (Force-Directed Graph, Constraint-Based Layout). As opposed to the Bubble Chart, which is mainly used for aggregated data, the Beeswarm can be used to display the actual data. It arranges single nodes based on selected properties, allowing to get insights about the distribution of the data.

Chart Options

width
Width of the chart.
Default: 600
height
Height of the chart.
Default: 600
top
Padding top.
Default: 30
bottom
Padding bottom.
Default: 80
left
Padding left.
Default: 200
right
Padding right.
Default: 80
nodeRadius
Radius of each node.
Default: 4
